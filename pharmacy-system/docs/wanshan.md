# 药店管理系统上线前完善方案

本文档采用方案 ：管理端可以开放访问，但只提供测试账号，并限制测试账号只能查看，不能新增、修改、删除真实数据。

## 目标

上线后访问者可以看到药店管理系统的完整界面和业务流程，但不能破坏数据库内容。

最终访问结构：

```text
www.yaoliumu.com
个人作品集、简历、项目展示、下载资源

admin.yaoliumu.com
药店管理系统管理端

client.yaoliumu.com
药店管理系统用户端

www.yaoliumu.com/api
Spring Boot 后端接口代理
```

## 管理端账号设计

保留真实管理员账号，仅自己使用：

```text
角色：ADMIN
权限：全部功能
用途：个人维护、演示前准备数据
```

新增演示账号：

```text
账号：demo
密码：上线前自行设置强密码
角色：DEMO
权限：只读
用途：给访问者或面试官查看管理端页面
```

不要在作品集页面直接公开管理员账号和密码。演示账号是否公开展示，可根据实际需要决定。

## DEMO 角色权限规则

DEMO 账号允许访问：

```text
登录
控制台
药品列表查看
供应商列表查看
顾客/会员列表查看
采购订单查看
采购明细查看
销售订单查看
销售明细查看
库存查看
库存盘点查看
系统日志查看
用户端相关展示数据查看
```

DEMO 账号禁止操作：

```text
新增
编辑
删除
批量删除
导入
导出敏感数据
修改库存
修改用户权限
重置密码
新增管理员
修改系统配置
清空日志
```

## 后端需要修改

### 1. 增加 DEMO 角色

数据库角色或用户表中需要支持：

```text
ADMIN
DEMO
USER
```

如果当前系统没有角色表，可以先在用户表中增加角色字段，例如：

```text
role
```

示例值：

```text
ADMIN
DEMO
USER
```

### 2. 接口权限校验

对所有管理端写操作增加权限判断。

写操作包括：

```text
POST
PUT
PATCH
DELETE
```

建议规则：

```text
ADMIN 可以执行全部操作
DEMO 只能执行 GET 查询操作
USER 不能访问管理端接口
```

当 DEMO 账号请求写接口时，后端返回：

```text
403 Forbidden
```

返回信息建议：

```text
演示账号仅支持查看，不能修改数据。
```

### 3. Spring Security 建议

管理端接口建议统一放在：

```text
/api/admin/**
```

用户端接口建议统一放在：

```text
/api/client/**
```

权限建议：

```text
/api/admin/** 需要 ADMIN 或 DEMO
/api/client/** 可以按当前业务决定是否公开
```

写操作额外判断：

```text
只有 ADMIN 可以执行
```

### 4. 防止前端绕过

不能只在前端隐藏按钮，后端必须拦截 DEMO 写操作。

原因：

```text
访问者可以直接调用接口。
如果后端不限制，只隐藏按钮没有安全意义。
```

## 管理端前端需要修改

### 1. 登录后识别角色

登录接口返回当前用户角色：

```json
{
  "username": "demo",
  "role": "DEMO"
}
```

前端保存角色信息后，用于控制界面显示。

### 2. DEMO 账号隐藏或禁用危险按钮

DEMO 登录后，以下按钮建议禁用或隐藏：

```text
新增
编辑
删除
批量删除
导入
导出
重置密码
修改权限
提交
保存
确认删除
```

推荐使用禁用状态，并显示提示：

```text
演示账号仅支持查看
```

这样访问者能理解功能存在，但不会误以为系统缺少这些能力。

### 3. 页面顶部增加演示模式提示

DEMO 登录后，管理端顶部显示：

```text
当前为演示账号，仅支持查看，不能修改数据。
```

提示样式建议使用浅黄色或浅绿色，不要遮挡主要操作区域。

### 4. 请求失败提示

如果 DEMO 账号调用写接口，前端收到 403 后显示：

```text
演示账号仅支持查看，不能修改数据。
```

不要显示后端异常堆栈或技术错误。

## 数据库准备

上线前建议准备演示数据，不使用私人或敏感数据。

演示数据应包括：

```text
药品数据
供应商数据
顾客/会员数据
采购订单
销售订单
库存记录
盘点记录
系统日志
```

注意：

```text
手机号、身份证、地址等字段使用虚拟数据。
不要上传真实客户信息。
不要上传真实采购或销售记录。
```

## 作品集页面展示建议

药店系统项目页可以这样写：

```text
管理端演示：admin.yaoliumu.com
演示账号仅支持查看，不开放新增、修改、删除权限。

用户端演示：client.yaoliumu.com
可查看药品查询、药店信息、会员中心和 AI 客服等功能。
```

如果暂时不想公开演示账号密码，可以写：

```text
管理端涉及后台数据维护权限，演示账号可在沟通时提供。
```

## 上线前测试清单

### ADMIN 账号测试

- 可以登录管理端。
- 可以新增药品。
- 可以编辑药品。
- 可以删除测试药品。
- 可以查看采购、销售、库存、日志等页面。
- 可以正常退出登录。

### DEMO 账号测试

- 可以登录管理端。
- 可以进入控制台。
- 可以查看所有列表页面。
- 不能新增数据。
- 不能编辑数据。
- 不能删除数据。
- 不能导入数据。
- 不能修改库存。
- 不能修改用户权限。
- 调用写接口时返回 403。
- 前端能显示友好提示。

### 用户端测试

- 首页可以打开。
- 药品查询正常。
- 药店信息正常。
- 会员中心正常。
- AI 客服正常。
- 用户端不会暴露管理端接口或管理员信息。

### 安全检查

- MySQL 不开放公网端口。
- Spring Boot 后端不直接暴露公网端口。
- 所有外部接口通过 Nginx 代理访问。
- 管理端写接口必须校验角色。
- DEMO 账号密码不能与 ADMIN 相同。
- ADMIN 密码不能写在前端代码或公开文档中。

## 推荐实施顺序

```text
1. 数据库增加 DEMO 角色或用户角色字段
2. 创建 demo 演示账号
3. 后端增加角色识别和写操作拦截
4. 管理端前端根据角色禁用危险按钮
5. 准备一批虚拟演示数据
6. 完整测试 ADMIN 和 DEMO 账号
7. 作品集页面补充演示账号说明
8. 上线后再次测试 admin.yaoliumu.com 和 client.yaoliumu.com
```

## 最低完成标准

上线前至少必须做到：

```text
DEMO 账号可以登录并查看页面
DEMO 账号不能执行新增、编辑、删除
后端对 DEMO 写操作返回 403
管理端前端显示演示账号只读提示
数据库中没有真实敏感数据
```
